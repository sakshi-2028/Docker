## 🧠 Quick Refresher

> **Docker** is a containerization tool that packages an app with all its dependencies into a single unit called a **container**, so it runs the same on any machine — dev, test, or production.

### Why Docker?
- 🚫 No more *"it works on my machine"*
- 🚀 Faster deploys & rollbacks
- 📦 Portable across environments
- 🧩 Easy to scale with microservices
- 💰 Lower infrastructure cost

### Docker vs Virtual Machine

| Docker | Virtual Machine |
|---|---|
| Lightweight | Heavy |
| Starts in seconds | Takes minutes |
| Shares host OS kernel | Has full OS |
| Fewer resources | More resources |
| Faster deployment | Slower |
